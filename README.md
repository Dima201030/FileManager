# fileMenager

Консольная система резервного копирования на `C++17`.

Программа следит за выбранной папкой, сохраняет версии файлов, помнит удалённые файлы `30` дней и позволяет восстановить их из бэкапа.

## Что умеет программа

- следить за одной исходной папкой;
- сохранять новые и изменённые файлы в бэкап;
- замечать изменения, сделанные при выключенной программе;
- не удалять файл из бэкапа сразу после удаления пользователем;
- хранить удалённый файл ещё `30` дней;
- восстанавливать файл из истории.
- полностью очищать весь бэкап одной командой;
- работать без консоли в постоянном фоновом режиме.

## Сборка

Из корня проекта:

```bash
cmake -S . -B build
cmake --build build
```

После сборки исполняемый файл будет здесь:

```bash
./build/fileMenager
```

## Запуск

```bash
./build/fileMenager
```

После запуска откроется консольный режим, в котором нужно вводить команды.

## Запуск без консоли

Если нужно, чтобы программа работала как фоновая служба, используй фоновый режим через ключ `--background`:

```bash
./build/fileMenager --background --source /путь/к/source --backup /путь/к/backup
```

Что делает:
- не ждёт команды из консоли;
- сам запускает мониторинг;
- работает постоянно, пока процесс не остановят;
- подходит для `launchd`, `systemd` и `Task Scheduler`.

Дополнительные параметры:

```bash
./build/fileMenager --background --source /путь/к/source --backup /путь/к/backup --scan-interval 10 --retention-days 30
```

Что делают параметры:
- `--background` включает фоновый режим;
- `--source` задаёт исходную папку;
- `--backup` задаёт папку бэкапа;
- `--scan-interval` задаёт интервал проверки в секундах;
- `--retention-days` задаёт срок хранения удалённых файлов в днях.

Если бэкап уже был настроен раньше и `metadata` уже существует, можно запускать только с `--backup`:

```bash
./build/fileMenager --background --backup /путь/к/backup
```

## Первая настройка и первая слежка

Ниже минимальный сценарий первого запуска.

### 1. Указать папку, за которой нужно следить

```text
set-source /путь/к/исходной_папке
```

Что делает:
устанавливает папку, файлы из которой будут резервироваться.

### 2. Указать папку для бэкапа

```text
set-backup /путь/к/папке_бэкапа
```

Что делает:
устанавливает папку, где будут храниться версии файлов и служебные метаданные.

### 3. Запустить слежку

```text
start
```

Что делает:
сразу выполняет первое сканирование и затем запускает фоновую слежку за папкой.

### 4. Проверить состояние

```text
status
```

Что делает:
показывает текущие папки, включена ли слежка, число активных файлов, удалённых файлов и задач в очереди.

### 5. Посмотреть активные файлы

```text
files
```

Что делает:
выводит список файлов, которые сейчас считаются актуальными в каталоге бэкапа.

## Пример первого запуска

```text
set-source /Users/dima/Documents/source
set-backup /Users/dima/Documents/backup
start
status
files
```

## Команды

| Команда | Что делает |
|---|---|
| `help` | Показать список всех команд |
| `status` | Показать текущую конфигурацию и состояние |
| `set-source [path]` | Установить исходную папку |
| `set-backup [path]` | Установить папку бэкапа |
| `start` | Запустить фоновую слежку |
| `stop` | Остановить фоновую слежку |
| `scan` | Один раз просканировать папку вручную |
| `files` | Показать активные файлы |
| `deleted` | Показать файлы, помеченные на удаление |
| `history <path>` | Показать историю версий файла |
| `restore <path>` | Восстановить файл из бэкапа |
| `cleanup` | Удалить из бэкапа файлы, у которых истёк срок хранения |
| `full-cleanup confirm` | Полностью очистить все версии, metadata и каталог бэкапа |
| `logs` | Показать последние сообщения журнала |
| `exit` | Корректно завершить программу |
